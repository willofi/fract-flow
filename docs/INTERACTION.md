# 🔗 인터랙션 모델

MindMap AI는 촉각적 정밀함을 최우선으로 생각합니다. 모든 상호작용은 인지 부하를 최소화하면서도 제어 능력을 최대화할 수 있도록 설계되었습니다.

---

## 📐 50/25/본체 분할 모델

**이동**, **크기 조절**, **연결** 간의 충돌을 해결하기 위해 노드의 인터랙션 영역을 정밀한 기하학적 영역으로 분할했습니다:

### 1. 연결 영역 (중앙 50%)
- **위치**: 상, 하, 좌, 우 각 모서리의 중앙 50% 구간.
- **트리거**: 마우스 근접 시 `handle-indicator-circle`이 나타납니다.
- **동작**: 이 원형 핸들을 클릭하거나 드래그하면 노드 연결이 시작됩니다.
- **커서**: 정밀한 연결을 나타내기 위해 `crosshair` (+) 커서가 표시됩니다.

### 2. 크기 조절 영역 (측면 25% + 코너)
- **위치**: 각 가장자리의 바깥쪽 25% 구간과 네 개의 모서리.
- **동작**: `NodeResizer`를 사용하여 노드의 크기를 동적으로 조절할 수 있습니다.
- **커서**: 조절 방향에 따른 화살표(↔, ↕, ↘)가 표시됩니다.

### 3. 이동 영역 (본체 내부)
- **위치**: 노드 내부의 전체 표면.
- **동작**: 클릭 시 노드가 선택되며, 드래그 시 이동합니다.
- **커서**: 이동 가능한 상태를 나타내는 `grab` 커서가 표시됩니다.

## 🎯 픽셀 단위의 제로 갭(Zero-Gap) 연결

노드 기반 UI에서 흔히 발생하는 시각적 문제 중 하나는 연결선이 노드 테두리에서 1-2픽셀 앞에서 멈추는 "플로팅 엣지(Floating Edge)" 현상입니다.

### 수학적 정렬
우리는 **초정밀 제로 차원 핸들(Pinpoint Zero-Dimension Handles)**을 통해 이를 해결했습니다:
- **크기**: 기능적 핸들의 크기를 `0px x 0px`로 설정했습니다.
- **포지셔닝**: 절대 오프셋(`top: 0`, `left: 50%`)을 사용하여 1px 두께의 테두리 위에 정확히 배치했습니다.
- **결과**: 목표 지점이 테두리 위의 0px 좌표이므로, 연결선 경로가 테두리 픽셀에서 정확히 종료되어 끊김 없는 건축학적 디자인을 완성합니다.

## 🖱️ 듀얼 모드 연결 지원

사용자의 작업 흐름에 따라 서로 다른 입력 방식을 지원합니다:

1.  **드래그 앤 드롭(Drag-and-Drop)**: 빠르고 유연한 "스케치" 작업에 적합합니다. 앵커를 잡아 다른 노드로 던지듯 연결합니다.
2.  **클릭 투 클릭(Click-to-Click)**: 고정밀 매핑에 적합합니다. 출발지 앵커를 클릭(활성화/강조 상태 진입)한 후 목적지 앵커를 클릭합니다. 드래그가 번거로운 대형 마인드맵이나 터치패드 환경에 이상적입니다.

## 🎨 우클릭 색상 커스터마이징

노드는 단순한 블록이 아닌 카테고리를 의미합니다.
- **동작**: 노드 위 어디서나 마우스 우클릭을 하여 **파스텔 팔레트**를 엽니다.
- **테마 적용**: 현재 라이트/다크 모드에 맞춰 색상이 동적으로 조정되어 대비와 가독성을 보장합니다.
